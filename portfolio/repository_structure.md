-   50,000+ PowerShell execution logs generated daily

-   Existing SIEM tools could not sufficiently reduce investigation

-   Developed Python/Hadoop analytics pipeline using linear regression

**LinkedIn post**

One of the most rewarding AI projects I\'ve worked on wasn\'t about
building the most sophisticated machine learning model---it was about
solving a real operational problem.

A cybersecurity team was generating more than **50,000 PowerShell
execution logs every day**. Existing SIEM tools could collect the data
but couldn\'t reduce the investigation workload to something analysts
could realistically manage.

Working with a member of the Red Team, we identified two behavioral
characteristics common to malicious PowerShell scripts: **script
length** and **whitespace density**.

Using those features, I developed a Python/Hadoop analytics workflow
that consistently reduced the daily investigation queue to approximately
**1--2 high-risk scripts**, allowing analysts to focus on the activity
most likely to represent malicious behavior.

The biggest lesson wasn\'t about linear regression---it was about the
value of combining domain expertise, explainable analytics, and
thoughtful feature engineering to solve enterprise-scale problems.

I\'ve documented the project in a new GitHub repository as part of my AI
leadership portfolio. It demonstrates how practical AI can augment
cybersecurity operations through transparent, measurable solutions.
